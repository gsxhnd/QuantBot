# 数据管理

## 1. 数据获取

### 1.1 数据源

| 市场 | 数据源 | 数据类型 |
|------|--------|---------|
| **美股** | yfinance | 行情、基本面 |
| **A股** | Qlib | 行情、因子 |
| **新闻** | Claw (OpenClaw/ZeroClaw) | 新闻文本、情绪分析 |

### 1.2 更新机制

- 行情数据：实时/分钟级更新
- 基本面数据：每日更新
- 新闻数据：Claw实时推送
- 自动重试和异常处理

### 1.3 Claw新闻数据

**新闻来源**（由Claw管理）：
- 财经媒体（Bloomberg、Reuters、财联社等）
- 官方公告（交易所公告、公司公告）
- 社交媒体（Twitter/X财经大V）

**推送类型**：
- 重要公告（财报、并购、分红）
- 行业动态（监管变化、竞争格局）
- 宏观经济（利率、政策、地缘政治）
- 持仓相关（持股公司突发消息）

## 2. 数据质量验证

### 2.1 市场数据校验

| 检查项 | 验证规则 |
|--------|---------|
| 完整性 | 无缺失交易日 |
| 异常值 | 价格/成交量在合理范围 |
| 重复数据 | 去重检查 |
| 时间顺序 | 严格递增 |
| 价格有效性 | 单日涨跌 < 阈值 |
| 成交量有效性 | 非零且无异常倍数 |

### 2.2 新闻数据处理

**质量验证**：
- 来源信誉度检查（Bloomberg、Reuters等可信源）
- 新鲜度检查（过期标记）
- 重复检查（TF-IDF相似度去重）
- 情绪分析置信度检查

**情绪分析格式**（由Claw提供）：

```json
{
  "news_id": "uuid",
  "title": "...",
  "source": "Reuters",
  "published_at": "2024-01-15T10:30:00Z",
  "related_stocks": ["AAPL", "MSFT"],
  "sentiment": {
    "overall": "bullish|bearish|neutral",
    "confidence": 0.85,
    "dimensions": {
      "financial_impact": {"score": 0.7, "direction": "positive"},
      "market_sentiment": {"score": 0.6, "direction": "positive"},
      "regulatory_risk": {"score": 0.1, "direction": "neutral"}
    },
    "impact_level": "high|medium|low",
    "expected_duration": "short|medium|long"
  },
  "summary": "..."
}
```

## 3. 数据存储

### 3.1 DuckDB方案

- 列式存储优化查询
- 时间戳和股票代码索引
- 保留2年历史数据
- 自动清理更早数据

### 3.2 冷热分离

- 热数据（近3月）：内存
- 温数据（3-12月）：SSD
- 冷数据（1-2年）：压缩存储

### 3.3 新闻数据存储

| 字段 | 类型 | 说明 |
|------|------|------|
| news_id | UUID | 唯一标识 |
| title | String | 标题 |
| source | String | 来源 |
| published_at | Timestamp | 发布时间 |
| related_stocks | Array | 相关股票 |
| sentiment_json | JSON | 情绪分析结果 |
| raw_content | String | 原文内容 |
| created_at | Timestamp | 入库时间 |

## 4. 数据生命周期

```
获取 → 验证 → 存储 → 使用 → 归档 → 清理
```

| 阶段 | 处理 |
|------|------|
| 获取 | 多源获取、自动重试 |
| 验证 | 质量检查、异常标记 |
| 存储 | 分类存储、索引优化 |
| 使用 | 特征计算、模型推理 |
| 归档 | 超期数据压缩存储 |
| 清理 | 超过保留期数据删除 |
