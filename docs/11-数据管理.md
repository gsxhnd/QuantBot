# 数据管理

## 1. 数据架构

### 1.1 数据分类

**市场数据**:

- 行情数据(OHLCV)
- 分钟级/日级数据
- 实时tick数据

**基本面数据**:

- 财务报表
- 公司公告
- 行业数据

**新闻数据**:

- 新闻标题和正文
- 发布时间和来源
- 情绪标签

**衍生数据**:

- 技术指标
- 因子数据
- 特征数据

### 1.2 数据存储

**DuckDB 表结构**:

```sql
-- 行情数据表
CREATE TABLE market_data (
    symbol VARCHAR,
    date DATE,
    open DECIMAL,
    high DECIMAL,
    low DECIMAL,
    close DECIMAL,
    volume BIGINT,
    PRIMARY KEY (symbol, date)
);

-- 技术指标表
CREATE TABLE indicators (
    symbol VARCHAR,
    date DATE,
    ma_5 DECIMAL,
    ma_20 DECIMAL,
    rsi DECIMAL,
    macd DECIMAL,
    PRIMARY KEY (symbol, date)
);

-- 新闻数据表
CREATE TABLE news (
    id VARCHAR PRIMARY KEY,
    symbol VARCHAR,
    title TEXT,
    content TEXT,
    source VARCHAR,
    publish_time TIMESTAMP,
    sentiment DECIMAL
);
```

## 3. 数据质量管理

### 3.1 数据完整性检查

**检查项目**:

```python
# 检查缺失值
缺失率 = 数据.isnull().sum() / len(数据)
if 缺失率 > 0.001:
    触发告警

# 检查时间连续性
日期差 = 数据['date'].diff()
if 日期差.max() > 5天:  # 考虑周末和节假日
    触发告警

# 检查数据量
if len(数据) < 预期数量 × 0.95:
    触发告警
```

### 3.2 数据异常检测

**价格异常**:

```python
# 单日涨跌幅检查
涨跌幅 = (收盘价 - 前收盘价) / 前收盘价

# A股涨跌停限制
if abs(涨跌幅) > 0.20:  # 超过20%
    标记异常

# 美股异常波动
if abs(涨跌幅) > 0.30:  # 超过30%
    标记异常
```

**成交量异常**:

```python
# 零成交量
if 成交量 == 0:
    标记异常

# 异常放量
平均成交量 = 过去20天平均成交量
if 成交量 > 平均成交量 × 5:
    标记异常
```

**价格有效性**:

```python
# 价格关系检查
if not (最低价 <= 开盘价 <= 最高价):
    标记异常
if not (最低价 <= 收盘价 <= 最高价):
    标记异常

# 价格为负或零
if 任何价格 <= 0:
    标记异常
```

### 3.3 数据重复检查

```python
# 检查重复记录
重复数据 = 数据[数据.duplicated(subset=['symbol', 'date'])]
if len(重复数据) > 0:
    去重并记录日志
```

### 3.4 数据一致性检查

**多源数据对比**:

```python
# 对比不同数据源
源1数据 = 从源1获取('AAPL', '2024-01-01')
源2数据 = 从源2获取('AAPL', '2024-01-01')

价格差异 = abs(源1数据.close - 源2数据.close) / 源1数据.close
if 价格差异 > 0.005:  # 差异超过0.5%
    触发告警
```

## 4. 新闻数据处理

### 4.1 新闻采集

**采集流程**:

1. 从RSS或API获取新闻
2. 解析标题和正文
3. 提取发布时间
4. 记录新闻来源

### 4.2 新闻质量验证

**来源验证**:

```python
可信来源列表 = [
    'Bloomberg',
    'Reuters', 
    'WSJ',
    'FT',
    '新华社',
    '证券时报'
]

if 新闻来源 in 可信来源列表:
    信誉度 = 1.0
else:
    信誉度 = 0.5
```

**新鲜度检查**:

```python
新闻年龄 = 当前时间 - 发布时间
if 新闻年龄 > 24小时:
    标记为过期
```

**重复检查**:

```python
# 基于TF-IDF相似度
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

相似度 = cosine_similarity(新新闻向量, 历史新闻向量)
if 相似度 > 0.8:
    标记为重复
```

### 4.3 情绪分析

**分析方法**:

- 基于词典的情绪分析
- 预训练模型(FinBERT等)
- 情绪分类(正面/中性/负面)

**置信度检查**:

```python
if 情绪置信度 < 0.6:
    标记为低置信度
    降低权重
```

## 5. 数据更新机制

### 5.1 自动更新流程

```python
# 每日更新任务
def 每日数据更新():
    # 1. 获取最新数据
    最新数据 = 从数据源获取(昨日日期)
    
    # 2. 数据质量检查
    if 数据质量检查(最新数据):
        # 3. 写入数据库
        写入数据库(最新数据)
        
        # 4. 更新技术指标
        更新技术指标()
        
        # 5. 记录日志
        记录更新日志()
    else:
        # 数据质量不合格
        触发告警()
```

### 5.2 更新时间

| 市场 | 更新时间 | 说明 |
|------|---------|------|
| A股 | 每日 16:00 | 收盘后 |
| 美股 | 每日 05:00 (北京时间) | 收盘后 |
| 新闻 | 每小时 | 实时更新 |

### 5.3 增量更新

```python
# 获取最后更新日期
最后日期 = 查询最后日期()

# 只获取新数据
新数据 = 获取数据(start=最后日期+1天, end=今天)

# 追加到数据库
追加数据(新数据)
```

## 6. 数据备份

### 6.1 备份策略

**全量备份**:

- 频率: 每周日
- 保留: 最近4周

**增量备份**:

## 7. 数据清理

### 7.1 历史数据保留

**保留策略**:

- 日线数据: 保留2年
- 分钟数据: 保留3个月
- 新闻数据: 保留1年
- 日志数据: 保留90天

### 7.2 自动清理

```python
# 清理过期数据
def 清理过期数据():
    截止日期 = 今天 - 保留天数
    
    DELETE FROM market_data 
    WHERE date < 截止日期
    
    记录清理日志()
```

## 9. 性能优化

### 9.1 索引优化

```sql
-- 创建索引加速查询
CREATE INDEX idx_symbol_date ON market_data(symbol, date);
CREATE INDEX idx_date ON market_data(date);
```

### 9.2 查询优化

```sql
-- 使用列式存储优势
SELECT symbol, close 
FROM market_data 
WHERE date BETWEEN '2024-01-01' AND '2024-12-31';

-- 避免SELECT *
```

### 9.3 数据分区

```sql
-- 按年份分区
CREATE TABLE market_data_2024 AS 
SELECT * FROM market_data WHERE year(date) = 2024;
```
