# 数据管理

## 1. 数据获取

### 1.1 数据源

| 市场 | 数据源 | 数据类型 |
|------|--------|---------|
| **美股** | yfinance | 行情、基本面 |
| **A股** | Qlib | 行情、因子 |
| **新闻** | RSS/API | 新闻文本 |

### 1.2 更新机制

- 行情数据：实时/分钟级更新
- 基本面数据：每日更新
- 新闻数据：实时抓取
- 自动重试和异常处理

## 2. 数据质量验证

### 2.1 市场数据校验

| 检查项 | 验证规则 |
|--------|---------|
| 完整性 | 无缺失交易日 |
| 异常值 | 价格/成交量在合理范围 |
| 重复数据 | 去重检查 |
| 时间顺序 | 严格递增 |
| 价格有效性 | 单日涨跌 < 阈值 |
| 成交量有效性 | 非零且无异常倍数 |

### 2.2 新闻数据处理

**质量验证**：
- 来源信誉度检查（Bloomberg、Reuters 等可信源）
- 新鲜度检查（过期标记）
- 重复检查（TF-IDF 相似度去重）
- 情绪分析置信度检查

## 3. 数据存储

### 3.1 DuckDB 方案

- 列式存储优化查询
- 时间戳和股票代码索引
- 保留 2 年历史数据
- 自动清理更早数据

### 3.2 冷热分离

- 热数据（近 3 月）：内存
- 温数据（3-12 月）：SSD
- 冷数据（1-2 年）：压缩存储
