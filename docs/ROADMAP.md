# QuantBot 项目路线图

## 项目概述

完全本地化的中长期量化分析系统，运行在 Jetson Orin Nano Super 平台，具备 7x24 小时市场监控、AI 驱动分析和投资建议能力。

**系统定位**：量化分析系统，提供投资建议，用户手动决策交易。

---

## Phase 1：核心系统搭建（第 1-2 月）

### 目标

建立基础数据流和模型推理能力

### 关键任务

#### 1.1 数据获取与存储

- [ ] 配置 yfinance（美股）和 Qlib（A 股）数据源
- [ ] 实现 DuckDB 数据库初始化和表结构设计
- [ ] 开发数据自动更新服务（定时任务）
- [ ] 实现数据持久化和索引优化

#### 1.2 特征工程

- [ ] 使用 Qlib 计算技术指标（MA、RSI、MACD 等）
- [ ] 实现特征标准化和归一化
- [ ] 构建训练数据集生成流程

#### 1.3 AI 模型训练

- [ ] 设计 GRU 网络架构（输入层、隐藏层、输出层）
- [ ] 实现 PyTorch 训练流程
- [ ] 配置早停机制和模型保存
- [ ] 导出 ONNX 格式模型
- [ ] 验证 ONNX 模型正确性

#### 1.4 Rust 推理服务

- [ ] 配置 ONNX Runtime (ort crate)
- [ ] 实现模型加载和推理接口
- [ ] 开发投资建议生成逻辑
- [ ] 构建基础 REST API（Axum 框架）

### 验收标准

- ✓ 能够获取并存储历史行情数据
- ✓ 模型训练完成并成功导出 ONNX
- ✓ Rust 服务能够加载模型并返回预测结果
- ✓ API 响应时间 < 2 秒

## Phase 2：风险管理框架（第 2-3 月）⚠️ P0 优先级

### 目标

建立完整的风险评估体系

### 关键任务

#### 2.1 风险指标计算

- [ ] 实现 VaR (95%) 计算（历史模拟法）

- [ ] 实现实时回撤跟踪

#### 2.2 仓位建议

- [ ] 实现 Kelly 准则仓位计算
- [ ] 提供单股敞口建议（< 5%）
- [ ] 提供总杠杆率建议（< 2.0）
- [ ] 开发行业敞口分散建议

#### 2.3 成本估算模型

- [ ] 实现佣金成本计算（分市场配置）
- [ ] 开发滑点估算模型（考虑波动率和时段）
- [ ] 构建流动性成本（Market Impact）模型
- [ ] 集成总成本估算

#### 2.4 监控告警系统

- [ ] 设计告警分级机制（INFO/WARNING/ERROR/CRITICAL）
- [ ] 实现策略监控（收益率、夏普比、胜率）
- [ ] 开发风险监控（回撤、VaR、杠杆率）
- [ ] 配置告警通知渠道（邮件）

### 验收标准

- ✓ VaR 突破时自动触发 CRITICAL 告警
- ✓ 仓位计算符合 Kelly 准则约束
- ✓ 成本估算在回测中准确应用

## 风险与应对

### 高优先级风险（P0）

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| **模型漂移未检测** | 建议失效，持续亏损 | Phase 4 必须完成漂移检测 |
| **风险控制缺失** | 无法评估风险，重大亏损 | Phase 2 优先实现 VaR 和仓位建议 |
| **内存溢出** | 系统崩溃 | Phase 6 优化内存管理 |

### 中等优先级风险（P1）

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| **数据源故障** | 无行情、无分析 | 实现多源备份和离线缓存 |
| **推理延迟过高** | 建议生成延迟 | 批量处理和硬件加速 |
| **滑点超预期** | 建议成本低估 | 完整成本估算模型 |

## 技术债务管理

### 必须避免的技术债

- ❌ 跳过风险管理直接上线
- ❌ 忽略模型漂移检测
- ❌ 回测中不考虑交易成本
- ❌ 不完善就上线

### 可接受的技术债

- ✓ Phase 1 使用简化的数据验证（Phase 4 完善）
- ✓ 初期使用固定策略权重（Phase 3 实现动态调整）
