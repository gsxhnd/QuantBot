# 多策略组合管理

## 1. 策略组合框架

### 1.1 组合管理理念

- **分散风险**: 不同策略相关性低,降低整体风险
- **平滑收益**: 多策略互补,减少收益波动
- **动态调整**: 根据表现动态分配权重
- **风险预算**: 每个策略分配独立风险预算

### 1.2 策略层级结构

```
第一层: 品种类策略
  ├─ 股票策略
  ├─ 期货策略
  └─ 期权策略
    ↓
第二层: 风格类策略
  ├─ 价值策略
  ├─ 成长策略
  ├─ 动量策略
  └─ 均值回归策略
    ↓
第三层: 具体策略
  ├─ GRU价格预测
  ├─ 技术指标组合
  ├─ 新闻情绪驱动
  └─ 因子选股
```

## 2. 权重分配方案

### 2.1 静态权重

**等权重配置**:

- 每个策略分配相同权重
- 简单易实现
- 适合策略表现相近的情况

**固定权重配置**:

- 根据历史表现手动设置
- 定期人工调整
- 适合策略特征明确的情况

### 2.2 动态权重(推荐)

**基于夏普比率的动态权重**:

```python
# 计算逻辑
过去N天夏普比率 = (策略收益率 - 无风险利率) / 策略波动率
归一化夏普比率 = max(0, 夏普比率)  # 负值设为0
原始权重 = 归一化夏普比率 / sum(所有策略归一化夏普比率)

# 应用约束
最终权重 = clip(原始权重, 最小权重, 最大权重)
重新归一化使权重和为1
```

**权重约束**:

- 最小权重: 10% (保持多样性)
- 最大权重: 40% (避免过度集中)
- 调整阈值: 变化 > 5% 才调整(减少交易)

### 2.3 风险平价

**风险贡献相等**:

```
每个策略的风险贡献 = 权重 × 波动率
调整权重使所有策略风险贡献相等
```

**优点**:

- 风险分散更均衡
- 不依赖收益预测
- 适合长期配置

## 3. 权重管理配置

### 3.1 更新频率

| 更新类型 | 频率 | 触发条件 |
|---------|------|---------|
| 日度微调 | 每日开盘前 | 常规更新 |
| 周度评估 | 每周一 | 绩效评估 |
| 月度调整 | 每月初 | 大幅调整 |
| 紧急调整 | 实时 | 风险事件 |

### 3.2 回看周期

- 短期: 5个交易日(捕捉近期表现)
- 中期: 20个交易日(平滑短期波动)
- 长期: 60个交易日(评估长期稳定性)

**推荐**: 使用20日作为主要回看周期

### 3.3 调整规则

**触发调整条件**:

- 权重变化 > 5%
- 策略夏普比 < 0.5 连续3天
- 策略回撤 > 15%
- 新策略上线

**调整限制**:

- 单次调整幅度 ≤ 10%
- 避免频繁调整(增加成本)
- 保持组合稳定性

## 4. 策略绩效评估

### 4.1 评估指标

**收益指标**:

- 累计收益率
- 年化收益率
- 月均收益率

**风险指标**:

- 波动率(标准差)
- 最大回撤
- 下行波动率

**风险调整收益**:

- 夏普比率
- 索提诺比率
- 卡玛比率

**其他指标**:

- 胜率
- 盈亏比
- 相关性矩阵

### 4.2 评估频率

**日度监控**:

- 当日收益率
- 累计收益
- 实时回撤

**周度评估**:

- 周收益和波动
- 夏普比率
- 与基准对比

**月度评估**:

- 月度绩效报告
- 策略排名
- 权重调整建议

### 4.3 策略排名

**综合评分**:

```
评分 = 0.4 × 夏普比率 + 0.3 × 收益率 + 0.3 × (1 - 最大回撤)
```

**排名用途**:

- 识别表现优异策略
- 发现表现不佳策略
- 指导权重调整

## 5. 策略生命周期管理

### 5.1 策略上线

**上线流程**:

1. 历史回测验证
2. 模拟盘测试(至少1个月)
3. 小仓位实盘测试(初始权重5%)
4. 表现稳定后逐步增加权重

**上线标准**:

- 回测夏普比 > 1.0
- 模拟盘表现符合预期
- 最大回撤 < 20%

### 5.2 策略运行

**正常运行**:

- 持续监控表现
- 定期评估调整
- 记录运行日志

**异常处理**:

- 表现异常时降低权重
- 严重异常时暂停策略
- 分析原因并修复

### 5.3 策略暂停

**暂停条件**:

- 连续3个月夏普比 < 0.5
- 单月回撤 > 25%
- 模型漂移严重
- 策略逻辑失效

**暂停流程**:

1. 降低权重至最小
2. 停止新开仓
3. 逐步平仓
4. 分析失效原因

### 5.4 策略重启

**重启条件**:

- 问题已修复
- 回测验证有效
- 模拟盘测试通过

**重启流程**:

1. 小仓位重新上线
2. 观察1-2周
3. 表现正常后恢复权重

## 6. 组合风险管理

### 6.1 相关性管理

**相关性监控**:

- 计算策略间收益相关性
- 目标: 相关性 < 0.5
- 高相关策略降低权重

**分散化指标**:

```
有效策略数 = 1 / sum(权重²)
目标: 有效策略数 > 3
```

### 6.2 组合风险预算

**风险分配**:

- 总风险预算: 账户资金 × 风险容忍度
- 策略风险预算: 总预算 × 策略权重
- 动态调整风险预算

### 6.3 组合再平衡

**再平衡触发**:

- 实际权重偏离目标 > 10%
- 定期再平衡(每月)

**再平衡方法**:

- 计算目标持仓
- 生成调仓订单
- 考虑交易成本
- 执行调仓

## 7. 组合绩效归因

### 7.1 收益归因

**分解组合收益**:

```
组合收益 = Σ(策略权重 × 策略收益)
```

**贡献分析**:

- 各策略收益贡献
- 识别主要收益来源
- 优化权重配置

### 7.2 风险归因

**风险分解**:

```
组合风险 = √(Σ Σ 权重i × 权重j × 协方差ij)
```

**风险贡献**:

- 各策略风险贡献
- 识别风险集中点
- 优化风险分散

### 7.3 超额收益归因

**Alpha分解**:

- 策略选择效应
- 权重配置效应
- 交互效应

## 8. 实施建议

### 8.1 初期配置

**建议起始配置**:

- 2-3个策略
- 等权重或略有倾斜
- 观察3个月后调整

### 8.2 扩展路径

**逐步增加策略**:

1. 验证核心策略稳定
2. 每次新增1个策略
3. 观察组合效果
4. 逐步优化权重

### 8.3 注意事项

- 避免过度优化(过拟合)
- 保持策略多样性
- 定期回顾和调整
- 记录决策过程
