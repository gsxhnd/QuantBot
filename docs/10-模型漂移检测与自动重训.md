# 模型漂移检测与自动重训

## 1. 模型漂移概述

### 1.1 什么是模型漂移

模型漂移是指模型在生产环境中的性能随时间逐渐下降的现象。

**主要原因**:
- 市场环境变化
- 数据分布变化
- 策略被市场学习
- 新的市场参与者

### 1.2 漂移的影响

- 预测准确率下降
- 策略收益降低
- 风险增加
- 信号质量变差

## 2. 漂移类型

### 2.1 数据漂移 (Data Drift)

**定义**: 输入特征的分布发生变化

**示例**:
- 市场波动率显著变化
- 成交量分布改变
- 价格区间迁移

### 2.2 特征漂移 (Feature Drift)

**定义**: 特征统计特性变化

**示例**:
- 技术指标均值偏移
- 特征相关性变化
- 特征重要性改变

### 2.3 概念漂移 (Concept Drift)

**定义**: 特征与目标变量的关系变化

**示例**:
- 原有规律失效
- 新的市场规律出现
- 因果关系改变

### 2.4 性能漂移 (Performance Drift)

**定义**: 模型预测性能下降

**示例**:
- 准确率下降
- 夏普比率降低
- Alpha衰减

## 3. 漂移检测方法

### 3.1 数据漂移检测

#### 3.1.1 Kolmogorov-Smirnov 检验

**原理**: 比较两个分布的累积分布函数

```python
from scipy.stats import ks_2samp

# 比较训练数据和生产数据
statistic, p_value = ks_2samp(训练数据, 生产数据)

if p_value < 0.05:
    # 分布显著不同,发生漂移
    触发告警
```

**阈值**: p-value < 0.05

#### 3.1.2 Population Stability Index (PSI)

**计算公式**:
```
PSI = Σ (实际占比 - 预期占比) × ln(实际占比 / 预期占比)
```

**判断标准**:
- PSI < 0.1: 无显著变化
- 0.1 ≤ PSI < 0.25: 轻微变化
- PSI ≥ 0.25: 显著变化,需重训

### 3.2 特征漂移检测

#### 3.2.1 Wasserstein 距离

**原理**: 衡量两个分布之间的距离

```python
from scipy.stats import wasserstein_distance

距离 = wasserstein_distance(训练分布, 生产分布)
基线距离 = 历史平均距离

if 距离 > 基线距离 × 1.5:
    触发告警
```

**阈值**: > 1.5倍基线

#### 3.2.2 特征统计监控

**监控指标**:
- 均值偏移
- 标准差变化
- 分位数变化
- 缺失值比例

### 3.3 概念漂移检测

#### 3.3.1 Alpha衰减监控

**计算方法**:
```python
当前Alpha = 当前策略收益 - 基准收益
历史Alpha = 过去N天平均Alpha

Alpha衰减率 = (历史Alpha - 当前Alpha) / 历史Alpha

if Alpha衰减率 > 0.3:
    触发概念漂移告警
```

**阈值**: 衰减 > 30%

#### 3.3.2 预测-实际偏离度

**监控方法**:
```python
预测收益率 = 模型预测
实际收益率 = 真实市场表现

偏离度 = abs(预测收益率 - 实际收益率)
平均偏离度 = 滑动窗口平均

if 偏离度 > 平均偏离度 × 2:
    触发告警
```

### 3.4 性能漂移检测

#### 3.4.1 滑动窗口性能监控

**监控指标**:
- 预测准确率
- 方向准确率
- AUC/ROC
- 夏普比率

**检测逻辑**:
```python
窗口大小 = 20天
当前窗口性能 = 最近20天性能
基线性能 = 训练时性能

性能下降率 = (基线性能 - 当前窗口性能) / 基线性能

if 性能下降率 > 0.15:
    触发性能漂移告警
```

**阈值**: 下降 > 15%

## 4. 漂移检测配置

### 4.1 检测频率

| 漂移类型 | 检测频率 | 数据窗口 |
|---------|---------|---------|
| 数据漂移 | 每日 | 最近20天 vs 训练期 |
| 特征漂移 | 每日 | 最近20天 vs 训练期 |
| 概念漂移 | 每周 | 最近60天 vs 训练期 |
| 性能漂移 | 每日 | 最近20天 vs 基线 |

### 4.2 告警阈值

| 检测方法 | 轻微漂移 | 中等漂移 | 严重漂移 |
|---------|---------|---------|---------|
| KS检验 p-value | 0.01-0.05 | 0.001-0.01 | < 0.001 |
| PSI | 0.1-0.25 | 0.25-0.5 | > 0.5 |
| 性能下降 | 10-15% | 15-25% | > 25% |
| Alpha衰减 | 20-30% | 30-50% | > 50% |

## 5. 自动重训机制

### 5.1 重训触发条件

**自动触发**:
- 中等或严重漂移检测
- 性能下降超过阈值
- 定期重训(每月)

**手动触发**:
- 市场环境重大变化
- 策略逻辑调整
- 新数据源加入

### 5.2 重训策略

#### 5.2.1 轻微漂移 - 参数微调

**方法**:
- 保留模型结构
- 加载预训练权重
- 使用小学习率微调
- 仅训练最后几层

**数据范围**: 最近20天

**频率**: 每周一次

#### 5.2.2 中等漂移 - 交叉验证重训

**方法**:
- 使用完整模型结构
- 从头开始训练
- 时间序列交叉验证
- 超参数保持不变

**数据范围**: 最近60天

**频率**: 检测到漂移时自动触发

#### 5.2.3 严重漂移 - 完全重训

**方法**:
- 重新评估模型架构
- 重新进行特征工程
- 超参数重新优化
- 完整的训练流程

**数据范围**: 全量历史数据

**频率**: 按需触发

### 5.3 重训流程

```python
# 1. 检测漂移
漂移类型, 漂移程度 = 检测漂移()

# 2. 确定重训策略
if 漂移程度 == "轻微":
    策略 = "参数微调"
    数据范围 = "最近20天"
elif 漂移程度 == "中等":
    策略 = "交叉验证重训"
    数据范围 = "最近60天"
else:
    策略 = "完全重训"
    数据范围 = "全量数据"

# 3. 准备数据
训练数据 = 加载数据(数据范围)

# 4. 训练新模型
新模型 = 训练(训练数据, 策略)

# 5. 验证新模型
验证结果 = 验证(新模型)

# 6. 决策部署
if 验证结果.性能 > 当前模型.性能:
    部署(新模型)
else:
    回滚()
    记录失败原因
```

## 6. 模型验证

### 6.1 样本外测试

**方法**:
- 使用最近未参与训练的数据
- 至少10个交易日
- 计算关键指标

**验证指标**:
- 预测准确率 > 基线
- 夏普比率 > 1.0
- 最大回撤 < 20%

### 6.2 A/B测试

**方法**:
- 新旧模型并行运行
- 分配不同资金比例
- 对比实际表现

**测试周期**: 2-4周

**决策标准**:
- 新模型显著优于旧模型
- 风险指标可接受
- 稳定性良好

### 6.3 压力测试

**测试场景**:
- 历史极端事件重放
- 假设情景测试
- 蒙特卡洛模拟

**通过标准**:
- 极端情况下不崩溃
- 最大回撤可控
- 风控机制有效

## 7. 模型版本管理

### 7.1 版本命名

**格式**: `model_v{major}.{minor}.{patch}_{date}`

**示例**: `model_v1.2.3_20260311`

**版本号规则**:
- major: 架构重大变更
- minor: 特征或参数调整
- patch: 微调或修复

### 7.2 版本记录

**记录内容**:
- 版本号和时间
- 训练数据范围
- 模型参数配置
- 训练指标
- 验证指标
- 变更说明

### 7.3 版本回滚

**回滚条件**:
- 新模型性能不达标
- 出现严重bug
- 风险指标异常

**回滚流程**:
1. 停止新模型
2. 加载上一版本
3. 验证恢复正常
4. 分析失败原因

## 8. 监控和告警

### 8.1 漂移监控仪表板

**展示内容**:
- 各类漂移指标趋势
- 模型性能曲线
- 重训历史记录
- 当前模型版本

### 8.2 告警配置

| 告警类型 | 触发条件 | 告警级别 |
|---------|---------|---------|
| 轻微漂移 | PSI > 0.1 | WARNING |
| 中等漂移 | PSI > 0.25 | ERROR |
| 严重漂移 | PSI > 0.5 | CRITICAL |
| 性能下降 | 下降 > 15% | ERROR |
| 重训失败 | 验证不通过 | ERROR |

### 8.3 定期报告

**周报**:
- 漂移检测结果
- 模型性能趋势
- 是否需要重训

**月报**:
- 重训历史总结
- 模型演进分析
- 优化建议
