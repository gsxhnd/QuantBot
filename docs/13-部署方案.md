# 部署方案

## 1. 本地部署（主要方案）

### 1.1 部署架构

- Rust服务作为系统服务运行
- Vue前端静态文件直接访问
- DuckDB直接文件存储
- 无容器依赖

### 1.2 启动方式

```bash
# 启动Rust API服务
systemctl start quant-api

# 访问前端
http://localhost:8080
```

### 1.3 Claw配置

```yaml
# config/claw.yaml
claw:
  type: openclaw  # 或 zeroclaw
  endpoint: http://claw-server:8080
  api_key: ${CLAW_API_KEY}
  
  # 新闻推送配置
  news_webhook: /api/claw/news
  
  # 通知配置
  notification:
    email: user@example.com
    push: true
```

## 2. Docker部署（可选）

### 2.1 适用场景

- 快速环境搭建
- 依赖隔离
- 备份迁移

### 2.2 架构特点

- 多阶段构建（Python训练 + Rust运行时）
- Docker Compose编排
- 数据卷持久化
- ARM64原生支持

### 2.3 目录结构

```
quant-system/
├── docker-compose.yml
├── Dockerfile
├── .env
└── data/
```

### 2.4 Docker Compose示例

```yaml
version: '3.8'
services:
  quantbot:
    build: .
    ports:
      - "8080:8080"
    volumes:
      - ./data:/app/data
      - ./config:/app/config
    environment:
      - CLAW_ENDPOINT=http://claw:8080
      - CLAW_API_KEY=${CLAW_API_KEY}
  
  claw:
    image: openclaw/openclaw:latest
    ports:
      - "8081:8080"
    volumes:
      - ./claw-config:/config
```

## 3. Jetson优化

### 3.1 内存管理

- 可用内存：6-7GB
- 模型量化减少体积
- 数据冷热分离
- 定期垃圾回收
- 服务分离运行（训练/推理不并行）

### 3.2 存储优化

- DuckDB列式压缩
- 索引优化
- 保留2年数据

### 3.3 计算优化

- CUDA加速推理
- 推理吞吐：100-200 QPS
- P99延迟：< 50ms
- 动态批处理

## 4. 性能目标

| 指标 | 目标值 |
|------|--------|
| 系统可用性 | 95% |
| 连续运行 | > 72h |
| API响应 | < 2s |
| 推理延迟 | < 100ms |

## 5. 网络配置

### 5.1 端口说明

| 端口 | 服务 | 说明 |
|------|------|------|
| 8080 | QuantBot API | 主服务端口 |
| 8081 | Claw | Claw服务端口（如本地部署） |

### 5.2 外部访问

- 仅本地访问，无公网暴露
- 数据通过Claw推送，无需入站连接
