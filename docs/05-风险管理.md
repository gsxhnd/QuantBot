# 风险管理

## 1. 风险管理框架

### 1.1 核心理念

- **事前控制**: 交易前风险评估,不符合条件不执行
- **事中监控**: 实时监控风险指标,及时预警
- **事后分析**: 复盘风险事件,优化风控策略

### 1.2 风险管理层级

```
第一层: 账户级风险控制(总资金、总杠杆)
  ↓
第二层: 策略级风险控制(单策略风险预算)
  ↓
第三层: 持仓级风险控制(单股票头寸限制)
```

## 2. 核心风险指标

### 2.1 风险指标定义

| 指标 | 定义 | 目标阈值 | 监控频率 |
|------|------|--------|--------|
| **VaR (95%)** | 95% 置信度下的最大亏损 | 单日亏损 < 账户 2% | 每分钟 |
| **最大回撤** | 峰值到谷底的最大跌幅 | < 20% | 每日 |
| **夏普比率** | 风险调整收益 | > 1.0 | 每日 |
| **卡玛比率** | 年收益 / 最大回撤 | > 0.5 | 每日 |
| **索提诺比率** | 下行风险调整收益 | > 1.5 | 每日 |
| **头寸集中度** | 单股票敞口 | < 5% | 每笔交易 |
| **总杠杆率** | 全部头寸 / 账户资产 | < 2.0 | 每笔交易 |
| **实时回撤** | 当前值与当日高点的差 | < 日均回撤 150% | 每分钟 |

### 2.2 VaR 计算方法

**历史模拟法**:

1. 收集过去 N 天的收益率数据
2. 按从小到大排序
3. 取第 5% 分位数作为 VaR

**参数法(假设正态分布)**:

```
VaR = μ - 1.645 × σ
其中 μ 为平均收益率, σ 为收益率标准差
```

**蒙特卡洛模拟法**:

1. 基于历史数据拟合分布
2. 随机模拟未来收益路径
3. 统计损失分布

## 3. 头寸管理

### 3.1 Kelly 准则

**最优头寸大小计算**:

```
f* = (p × b - q) / b

其中:
f* = 最优投资比例
p = 胜率
q = 败率 (1-p)
b = 盈亏比 (平均盈利/平均亏损)
```

**实际应用**:

- 使用 Kelly 准则的 1/4 或 1/2(保守)
- 避免过度集中
- 考虑相关性

### 3.2 头寸约束

#### 3.2.1 单股票限制

- 单股票头寸 ≤ 账户资金 5%
- 高波动股票进一步降低限制
- 考虑流动性因素

#### 3.2.2 行业限制

- 单行业敞口 ≤ 账户资金 20%
- 避免行业集中风险
- 动态调整行业配置

#### 3.2.3 总杠杆限制

- 总杠杆率 ≤ 2.0
- 市场波动加大时降低杠杆
- 保留安全边际

### 3.3 动态调整

**根据市场环境调整**:

- 高波动期: 降低头寸,降低杠杆
- 低波动期: 适当增加头寸
- 趋势明确: 增加趋势方向头寸

## 4. 止损止盈策略

### 4.1 止损设置

#### 4.1.1 固定止损

- 单笔交易最大亏损 2%
- 触发立即平仓
- 无条件执行

#### 4.1.2 移动止损

- 盈利后上移止损位
- 保护已有利润
- 让利润奔跑

#### 4.1.3 时间止损

- 持仓超过 N 天未盈利
- 自动平仓
- 避免资金占用

### 4.2 止盈设置

#### 4.2.1 目标止盈

- 达到预期收益率平仓
- 如 5%, 10%
- 落袋为安

#### 4.2.2 分批止盈

- 盈利后分批减仓
- 保留部分仓位
- 平衡收益和风险

## 5. 风险预警和应急响应

### 5.1 告警级别

#### 5.1.1 WARNING 级别

**触发条件**:

- 夏普比 < 1.0
- 单日回撤 > 3%
- 连续 3 天负收益

**响应动作**:

- 邮件通知
- 记录日志
- 人工关注

#### 5.1.2 ERROR 级别

**触发条件**:

- 实时回撤 > 日均 150%
- 头寸集中度 > 10%
- 杠杆率 > 3.0

**响应动作**:

- 即时推送通知
- 仪表板高亮显示
- 模拟盘自动减仓 50%

#### 5.1.3 CRITICAL 级别

**触发条件**:

- VaR 突破阈值
- 单日亏损 > 5%
- 系统异常

**响应动作**:

- 紧急推送通知
- 自动平仓对冲
- 停止所有交易
- 人工介入

### 5.2 应急预案

#### 5.2.1 市场极端波动

1. 立即停止新开仓
2. 评估现有持仓风险
3. 对冲或减仓
4. 等待市场稳定

#### 5.2.2 系统故障

1. 切换到备用系统
2. 手动监控持仓
3. 必要时手动平仓
4. 修复系统后恢复

#### 5.2.3 数据异常

1. 停止依赖异常数据的交易
2. 切换备用数据源
3. 验证数据质量
4. 恢复后重启

## 6. 风险报告

### 6.1 日报内容

- 当日收益率
- 当日最大回撤
- 当前持仓风险
- VaR 值
- 风险事件记录

### 6.2 周报内容

- 周收益率和波动率
- 夏普比、索提诺比
- 风险指标趋势
- 异常事件分析

### 6.3 月报内容

- 月度绩效总结
- 风险收益分析
- 最大回撤分析
- 风控策略优化建议
