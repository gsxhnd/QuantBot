# 回测框架

## 1. 回测原则

- **避免前瞻偏差**：时间序列交叉验证，严格时间顺序
- **真实性**：完整成本、真实滑点、流动性约束
- **验证目的**：验证策略有效性，而非保证未来收益

## 2. 回测方法

### 2.1 Walk-Forward分析

```
训练窗口: 252天（1年）
测试窗口: 63天（3个月）
滚动步长: 21天（1个月）

[训练1][测试1]
    [训练2][测试2]
        [训练3][测试3]
```

### 2.2 蒙特卡洛模拟

- 随机打乱交易顺序
- 模拟1000次
- 评估收益分布和尾部风险

### 2.3 压力测试

重放历史极端事件：
- 2008金融危机
- 2020疫情暴跌
- 2015 A股股灾

## 3. 评估指标

### 3.1 收益指标

- 总收益率、年化收益率
- 月收益波动率

### 3.2 风险指标

- 夏普比、索提诺比、卡玛比
- 最大回撤
- 95% VaR、CVaR

### 3.3 交易指标

- 胜率、平均胜负比
- 盈利因子
- 最大连续亏损笔数

### 3.4 基准对比

- 超额收益
- 信息比
- Beta、Alpha

## 4. 成本模拟

回测中应用完整成本模型：
1. 佣金成本
2. 滑点成本（考虑时段、波动率）
3. 流动性成本
4. 融资融券成本

**净回报** = 策略回报 - 总成本

## 5. 报告生成

- **日报**：收益率、胜率、回撤
- **周报**：夏普比、信息比、因子归因
- **月报**：完整绩效、风险统计、基准对比

## 6. 回测局限性说明

**重要提示**：

1. 回测结果不代表未来表现
2. 过度拟合风险：策略可能在历史数据上表现良好但未来失效
3. 市场环境变化：历史条件可能不再适用
4. 执行差异：实际交易可能与回测假设不同
5. 用户应结合自身判断做出投资决策
