# AI模型设计

## 1. 模型架构

**Python/Rust混合架构**

- Python：数据获取、特征工程、训练、ONNX导出
- Rust：模型加载、实时推理、信号生成

**GRU时序模型**

- 适合时序预测，比LSTM更轻量
- Jetson推理性能好
- 捕捉长期依赖

## 2. 训练流程

### 2.1 数据准备

**数据源**：yfinance（美股）、Qlib（A股），至少2年历史

**特征工程**：

- 技术指标：MA、EMA、RSI、MACD、Bollinger Bands
- 价格特征：收益率、波动率、振幅
- 成交量特征：成交量变化率、量价关系
- 时间特征：星期、月份、季度
- 新闻情绪特征：情绪得分、影响程度（来自Claw）

### 2.2 模型配置

**结构**：GRU(input → hidden_size=128, num_layers=2, dropout=0.2) → Linear(1)

**超参数**：

- 序列长度：20天
- 学习率：0.001
- 批大小：64
- 优化器：Adam + 学习率衰减
- 早停：验证集损失10 epoch不降

**数据划分**：训练70%、验证15%、测试15%

**评估指标**：MAE、RMSE、R²、方向准确率

## 3. 推理实现

**Rust推理服务**（依赖：ort、ndarray）

**流程**：

1. 加载ONNX模型（配置线程、启用CUDA）
2. 接收实时特征数据
3. 预处理和标准化
4. 执行推理
5. 生成投资建议

## 4. 信号生成逻辑

### 4.1 基础信号

- 预测收益率 > 阈值 → 买入建议
- 预测收益率 < -阈值 → 卖出建议
- 其他 → 持有/观望

### 4.2 置信度计算

```python
confidence = base_confidence * model_confidence * news_sentiment_factor
```

| 因子 | 权重 | 说明 |
|------|------|------|
| 模型置信度 | 0.5 | 预测概率/历史准确率 |
| 新闻情绪因子 | 0.3 | 正面情绪增强，负面减弱 |
| 市场状态因子 | 0.2 | 波动率、趋势强度 |

## 7. 新闻情绪集成

### 7.1 情绪因子使用

| 情绪维度 | 使用方式 |
|---------|---------|
| 整体情绪 | 调整信号方向/强度 |
| 置信度 | 调整信号置信度 |
| 影响程度 | 调整目标价/止损价 |
| 持续时间 | 调整持仓建议周期 |

### 7.2 情绪与价格信号融合

```python
final_signal = price_signal * (1 + sentiment_adjustment)
```

- 正面情绪：增强买入信号，提高置信度
- 负面情绪：减弱买入信号或转为卖出建议
- 中性情绪：保持价格信号不变
