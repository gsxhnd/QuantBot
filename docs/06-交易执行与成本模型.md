# 交易执行与成本模型

## 1. 交易成本模型

**总成本** = 佣金 + 滑点 + 流动性成本

### 1.1 佣金成本

| 市场 | 费用明细 |
|------|---------|
| **A股** | 买卖佣金 0.03%（最低5元）+ 印花税 0.1%（卖出）+ 过户费 0.001% |
| **美股** | 佣金 $0.005/股 + SEC费用 $0.0000278/美元 |

### 1.2 滑点模型

**公式**：滑点 = 基础滑点 × 波动率系数 × 时段系数

- 基础滑点：0.1% - 0.3%
- 波动率系数：当前波动率 / 历史平均
- 时段系数：开盘/收盘 1.5，盘中 1.0

### 1.3 流动性成本

**Market Impact**：流动性成本 = α × (订单量 / 日均成交量)^β

- 高流动性：α=0.1, β=0.5
- 中流动性：α=0.3, β=0.6
- 低流动性：α=0.5, β=0.7

## 2. 交易执行策略

### 2.1 交易前检查

| 检查项 | 标准 |
|--------|------|
| 流动性 | 订单量 ≤ 日均成交 10% |
| 价差 | 买卖价差 > 0.5% 时警告 |
| 账户 | 余额充足、无停牌 |
| 权限 | 支持做空/融资融券 |

### 2.2 订单拆分

| 订单类型 | 执行策略 |
|---------|---------|
| 小单（< 1%） | 直接市价成交 |
| 中单（1%-10%） | 分 3-5 批 TWAP |
| 大单（> 10%） | 分 10+ 批 VWAP |

## 3. 容错机制

### 3.1 失败场景处理

| 场景 | 容错策略 |
|------|---------|
| 委托被拒 | 预检查 + 重试 3 次 |
| 成交延迟 | 超时重发 + 本地队列 |
| 部分成交 | 自动拆分 + 递进成交 |
| 确认丢失 | 定期对账 + 重复保护 |
| 系统崩溃 | 快照恢复（RPO 24h） |

### 3.2 对账机制

- 定期与券商对账
- 比较本地与券商订单
- 处理状态不一致
- 检查未知订单
