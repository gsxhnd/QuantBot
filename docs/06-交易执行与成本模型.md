# 交易执行与成本模型

## 1. 交易成本模型

### 1.1 成本组成

交易总成本 = 佣金成本 + 滑点成本 + 流动性成本

### 1.2 佣金成本

#### 1.2.1 A股市场

- 买入佣金: 0.03% (最低5元)
- 卖出佣金: 0.03% (最低5元)
- 印花税: 0.1% (仅卖出)
- 过户费: 0.001%

#### 1.2.2 美股市场

- 佣金: $0.005/股 或固定费用
- SEC费用: $0.0000278/美元成交额
- 无印花税

### 1.3 滑点模型

**滑点定义**: 预期成交价与实际成交价的差异

**影响因素**:

- 市场波动率
- 订单大小
- 交易时段
- 流动性

**计算公式**:

```
滑点 = 基础滑点 × 波动率系数 × 时段系数

基础滑点: 0.1% - 0.3%
波动率系数: 当前波动率 / 历史平均波动率
时段系数: 开盘/收盘 1.5, 盘中 1.0
```

### 1.4 流动性成本

**Market Impact 模型**:

```
流动性成本 = α × (订单量 / 日均成交量)^β

α: 流动性系数 (0.1 - 0.5)
β: 非线性系数 (0.5 - 0.7)
```

**流动性分级**:

- 高流动性: 日均成交 > 1亿, α = 0.1
- 中流动性: 日均成交 1000万-1亿, α = 0.3
- 低流动性: 日均成交 < 1000万, α = 0.5

## 2. 交易前检查

### 2.1 流动性检查

**检查规则**:

- 订单量 ≤ 日均成交量 10%
- 超过阈值则拆单或放弃

**流动性评估**:

- 计算过去20天平均成交量
- 评估买卖价差
- 检查盘口深度

### 2.2 价差检查

**检查规则**:

- 买卖价差 > 0.5% 时发出警告
- 价差 > 1% 时建议延迟交易

### 2.3 账户检查

**检查项目**:

- 可用资金是否充足
- 是否有足够持仓可卖
- 标的是否停牌
- 是否在交易时间

### 2.4 权限检查

**检查项目**:

- 标的是否支持交易
- 是否有做空权限(融券)
- 是否有融资权限
- 风控限制检查

## 3. 订单执行策略

### 3.1 订单分类

#### 3.1.1 小单 (< 日均成交1%)

**执行策略**: 直接市价单

- 立即成交
- 滑点可控
- 简单高效

#### 3.1.2 中单 (1%-10%)

**执行策略**: TWAP (时间加权平均价格)

- 分 3-5 批执行
- 均匀分布在时间段内
- 减少市场冲击

#### 3.1.3 大单 (> 10%)

**执行策略**: VWAP (成交量加权平均价格)

- 分 10+ 批执行
- 根据历史成交量分布
- 最小化市场冲击

### 3.2 订单拆分

**拆分原则**:

- 单批不超过日均成交 1%
- 时间间隔 1-5 分钟
- 根据盘口动态调整

**拆分算法**:

```
总订单量 = N
日均成交量 = V
单批大小 = min(N/10, V × 1%)
批次数 = ceil(N / 单批大小)
```

### 3.3 执行时机

**最佳执行时段**:

- 避开开盘前30分钟(波动大)
- 避开收盘前30分钟(波动大)
- 优选盘中流动性好的时段

**紧急执行**:

- 风控平仓: 立即执行
- 止损单: 立即执行
- 普通单: 择机执行

## 4. 执行容错机制

### 4.1 常见失败场景

| 失败场景 | 原因 | 容错策略 |
|---------|------|---------|
| 委托被拒 | 余额不足/停牌/权限 | 预检查 + 重试3次 |
| 成交延迟 | 网络/交易所繁忙 | 超时重发 + 本地队列 |
| 价格偏离 | 行情变化/滑点 | 限价单 + 有效期 |
| 部分成交 | 流动性不足 | 自动拆分 + 递进成交 |
| 确认丢失 | 网络中断 | 定期对账 + 去重 |
| 系统崩溃 | 进程异常/OOM | 快照恢复 |
| API故障 | 第三方问题 | 多源备份 + 切换 |

### 4.2 重试机制

**指数退避重试**:

```
第1次: 立即重试
第2次: 等待 1秒
第3次: 等待 2秒
第4次: 等待 4秒
最多重试 5次
```

**重试条件**:

- 网络超时: 重试
- 服务器错误: 重试
- 参数错误: 不重试
- 权限错误: 不重试

### 4.3 订单状态管理

**订单状态流转**:

```
待提交 → 已提交 → 部分成交 → 全部成交
                ↓
              已撤销/失败
```

**状态持久化**:

- 每个状态变更写入数据库
- 支持断点恢复
- 防止重复提交

## 5. 对账机制

### 5.1 实时对账

**对账频率**: 每笔交易后立即对账

**对账内容**:

- 订单状态一致性
- 成交数量一致性
- 成交价格合理性
- 持仓数量一致性

### 5.2 定期对账

**对账频率**: 每日收盘后

**对账流程**:

1. 获取券商端持仓和订单
2. 对比本地记录
3. 标记差异项
4. 生成对账报告
5. 人工确认差异

### 5.3 差异处理

**状态不一致**:

- 本地已提交,券商未收到: 重新提交
- 本地未成交,券商已成交: 更新本地状态
- 记录异常日志

**未知订单**:

- 券商有订单,本地无记录: 告警并记录
- 可能是手动操作或系统遗漏

## 6. 成本在回测中的应用

### 6.1 回测成本计算

**每笔交易成本**:

```python
佣金 = max(成交金额 × 佣金率, 最低佣金)
滑点 = 成交金额 × 滑点率
流动性成本 = 成交金额 × 流动性系数
总成本 = 佣金 + 滑点 + 流动性成本
```

### 6.2 净收益计算

```
策略总收益 = 所有交易盈亏之和
总成本 = 所有交易成本之和
净收益 = 策略总收益 - 总成本
净收益率 = 净收益 / 初始资金
```

### 6.3 成本敏感性分析

**分析维度**:

- 不同佣金率下的策略表现
- 不同滑点假设下的策略表现
- 交易频率对成本的影响
- 优化交易频率

## 7. 执行性能监控

### 7.1 监控指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 委托成功率 | > 99% | 提交成功比例 |
| 成交确认延迟 | < 2秒 | 确认时间 |
| 实际滑点 | < 预期滑点 | 执行质量 |
| 部分成交率 | < 5% | 流动性指标 |
| 撤单率 | < 2% | 执行效率 |

### 7.2 执行质量评估

**VWAP 偏离度**:

```
偏离度 = (实际成交均价 - VWAP) / VWAP
目标: 偏离度 < 0.5%
```

**实施缺口**:

```
实施缺口 = 决策价 - 实际成交价
分析原因: 延迟、滑点、市场冲击
```
