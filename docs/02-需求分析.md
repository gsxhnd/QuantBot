# 需求分析

## 1. 功能需求

### 1.1 核心功能

#### 1.1.1 市场监控

- 7x24 小时实时盯盘
- 多市场支持(A股、美股)
- 行情数据实时更新
- 异常波动预警

#### 1.1.2 数据分析

- 历史数据获取和存储
- 技术指标计算
- 新闻数据采集
- 情绪分析处理

#### 1.1.3 交易决策

- 基于 AI 模型的价格预测
- 买卖信号生成
- 交易时机判断
- 风险收益评估

### 1.2 辅助功能

#### 1.2.1 账户管理

- 模拟盘交易
- 实盘交易支持
- 持仓管理
- 资金管理

#### 1.2.2 风险管理

- VaR (Value at Risk) 计算
- 头寸大小限制
- 最大回撤控制
- 杠杆率限制
- 止损止盈设置

#### 1.2.3 策略管理

- 多策略组合
- 策略权重分配
- 动态权重调整
- 策略绩效评估
- 策略版本管理

#### 1.2.4 回测系统

- 历史数据回测
- 交易成本模拟(佣金、滑点、流动性成本)
- 回测报告生成
- 可视化分析

#### 1.2.5 模型管理

- 模型训练
- 模型评估
- 模型部署
- 模型漂移检测
- 自动重训练

#### 1.2.6 监控告警

- 数据源监控
- 模型性能监控
- 系统资源监控
- 交易执行监控
- 风险指标监控
- 多级告警通知

#### 1.2.7 数据质量

- 数据完整性检查
- 异常值检测
- 数据一致性验证
- 新闻数据质量验证

#### 1.2.8 交易执行

- 订单提交
- 成交确认
- 执行容错
- 自动对账
- 订单拆分优化

## 2. 非功能需求

### 2.1 性能要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| API 响应时间 | < 2 秒 | 用户操作响应 |
| 模型推理延迟 | < 100ms | 单笔预测延迟 |
| 推理吞吐量 | > 100 次/秒 | 支持批量处理 |
| 数据处理量 | 10 万条/天 | 行情和新闻数据 |
| 并发用户数 | 1 人 | 单用户系统 |

### 2.2 安全要求

#### 2.2.1 数据安全

- 完全本地运行,数据不出本地
- 敏感信息加密存储
- API 密钥安全管理

#### 2.2.2 网络安全

- 无外部网络依赖(除数据获取外)
- 数据传输加密
- 访问权限控制

### 2.3 可用性要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 系统可用性 | 95% | 考虑 Jetson 硬件限制 |
| 恢复时间目标(RTO) | 1 小时 | 故障后恢复时间 |
| 恢复点目标(RPO) | 24 小时 | 最大数据丢失时间 |
| 连续运行时间 | > 72 小时 | 无故障运行 |

### 2.4 可维护性要求

- 模块化设计,易于扩展
- 完善的日志记录
- 清晰的代码文档
- 自动化测试覆盖
- 配置文件管理

### 2.5 可扩展性要求

- 支持新增交易市场
- 支持新增交易策略
- 支持新增数据源
- 支持新增 AI 模型

## 3. 约束条件

### 3.1 硬件约束

- 内存限制: 8GB(实际可用 6-7GB)
- 存储空间: 需预留足够空间存储历史数据
- 计算能力: 受 Jetson 性能限制

### 3.2 软件约束

- 必须支持 ARM64 架构
- 必须支持 Linux 系统
- 依赖开源免费软件

### 3.3 数据约束

- 依赖免费数据源(yfinance、Qlib)
- 数据延迟受数据源限制
- 历史数据有限

## 4. 用户场景

### 4.1 策略研发

1. 用户设计交易策略
2. 使用历史数据回测
3. 分析回测结果
4. 优化策略参数
5. 部署到模拟盘测试

### 4.2 实盘交易

1. 系统实时监控市场
2. AI 模型生成交易信号
3. 风险管理模块评估
4. 自动执行交易
5. 实时监控持仓和风险

### 4.3 风险监控

1. 系统计算风险指标
2. 触发风险预警
3. 用户收到告警通知
4. 系统自动执行风控措施
5. 记录风险事件

### 4.4 模型维护

1. 系统检测模型漂移
2. 触发自动重训练
3. 验证新模型性能
4. 自动部署新模型
5. 记录模型版本
