# 需求分析

## 1. 功能需求

### 1.1 核心功能

| 功能模块 | 功能描述 |
|---------|---------|
| **市场监控** | 7x24 盯盘、多市场支持、异常波动预警 |
| **数据分析** | 历史数据获取、技术指标计算、新闻情绪分析 |
| **交易决策** | AI 价格预测、买卖信号生成、风险收益评估 |

### 1.2 辅助功能

| 功能模块 | 核心能力 |
|---------|---------|
| **账户管理** | 模拟盘/实盘、持仓管理、资金管理 |
| **风险管理** | VaR 计算、头寸限制、回撤控制、止损止盈 |
| **策略管理** | 多策略组合、权重分配、绩效评估、版本管理 |
| **回测系统** | 历史回测、成本模拟、报告生成、可视化 |
| **模型管理** | 训练评估、漂移检测、自动重训 |
| **监控告警** | 数据/模型/系统/交易/风险监控、多级告警 |
| **数据质量** | 完整性检查、异常检测、一致性验证 |
| **交易执行** | 订单提交、成交确认、执行容错、自动对账 |

## 2. 非功能需求

### 2.1 性能要求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| API 响应时间 | < 2 秒 | 用户操作响应 |
| 模型推理延迟 | < 100ms | 单笔预测延迟 |
| 推理吞吐量 | > 100 次/秒 | 支持批量处理 |
| 数据处理量 | 10 万条/天 | 行情和新闻数据 |
| 并发用户数 | 1 人 | 单用户系统 |

### 2.2 安全要求

#### 2.2.1 数据安全

- 完全本地运行,数据不出本地
- 敏感信息加密存储
- API 密钥安全管理

#### 2.2.2 网络安全

- 无外部网络依赖(除数据获取外)
- 数据传输加密
- 访问权限控制

### 2.3 可用性要求

| 指标 | 目标值 |
|------|--------|
| 系统可用性 | 95% |
| 恢复时间目标(RTO) | 1 小时 |
| 恢复点目标(RPO) | 24 小时 |
| 连续运行时间 | > 72 小时 |

## 3. 约束条件

| 约束类型 | 具体限制 |
|---------|---------|
| **硬件** | 内存 8GB（可用 6-7GB）、ARM64 架构、Jetson 性能限制 |
| **软件** | 必须支持 Linux、依赖开源免费软件 |
| **数据** | 免费数据源（yfinance、Qlib）、数据延迟、历史数据有限 |
