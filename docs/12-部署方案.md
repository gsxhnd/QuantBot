# 部署方案

## 1. 部署架构

### 1.1 本地部署(推荐)

**架构特点**:

- 直接运行二进制文件
- 无容器依赖
- 资源占用最小
- 启动速度快

**适用场景**:

- 日常使用
- 性能要求高
- 资源受限环境

### 1.2 Docker部署(可选)

**架构特点**:

- 环境隔离
- 依赖管理简单
- 便于迁移
- 支持多实例

**适用场景**:

- 快速环境搭建
- 测试和开发
- 备份和迁移

## 2. 本地部署

### 2.1 系统要求

**硬件要求**:

- CPU: ARM Cortex-A78AE 6核或更高
- 内存: 8GB
- 存储: 50GB可用空间
- GPU: NVIDIA Ampere架构(可选)

**软件要求**:

- 操作系统: Ubuntu 20.04/22.04 (ARM64)
- Python: 3.10+
- Rust: 1.70+
- Node.js: 18+ (前端构建)

### 2.2 安装步骤

#### 2.2.1 安装系统依赖

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装基础工具
sudo apt install -y build-essential git curl

# 安装Python
sudo apt install -y python3.10 python3-pip

# 安装Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env

# 安装Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
```

#### 2.2.2 克隆项目

```bash
git clone https://github.com/your-repo/quant-system.git
cd quant-system
```

#### 2.2.3 安装Python依赖

```bash
cd python
pip3 install -r requirements.txt
```

#### 2.2.4 编译Rust服务

```bash
cd ../rust
cargo build --release
```

#### 2.2.5 构建前端

```bash
cd ../frontend
npm install
npm run build
```

### 2.3 配置

#### 2.3.1 创建配置文件

```bash
# 复制配置模板
cp config/config.example.yaml config/config.yaml

# 编辑配置
vim config/config.yaml
```

#### 2.3.2 配置示例

```yaml
# 数据库配置
database:
  path: ./data/quant.duckdb

# 数据源配置
data_sources:
  yfinance:
    enabled: true
    proxy: null
  qlib:
    enabled: true
    data_path: ./data/qlib

# API配置
api:
  host: 0.0.0.0
  port: 8080

# 模型配置
model:
  path: ./models/model.onnx
  device: cuda  # 或 cpu

# 风险管理配置
risk:
  max_position_size: 0.05
  max_leverage: 2.0
  var_threshold: 0.02
```

### 2.4 启动服务

#### 2.4.1 启动Rust API服务

```bash
cd rust
./target/release/quant-api
```

#### 2.4.2 启动前端服务

```bash
cd frontend
npm run preview
```

#### 2.4.3 使用systemd管理服务

```bash
# 创建服务文件
sudo vim /etc/systemd/system/quant-api.service
```

```ini
[Unit]
Description=Quant Trading API Service
After=network.target

[Service]
Type=simple
User=your-user
WorkingDirectory=/path/to/quant-system/rust
ExecStart=/path/to/quant-system/rust/target/release/quant-api
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

```bash
# 启动服务
sudo systemctl daemon-reload
sudo systemctl enable quant-api
sudo systemctl start quant-api

# 查看状态
sudo systemctl status quant-api
```

## 3. Docker部署

### 3.1 目录结构

```
quant-system/
├── docker-compose.yml
├── Dockerfile.api
├── Dockerfile.training
├── .env
└── data/
```

### 3.2 Dockerfile

#### 3.2.1 API服务镜像

```dockerfile
# Dockerfile.api
FROM rust:1.70 as builder

WORKDIR /app
COPY rust/ .
RUN cargo build --release

FROM ubuntu:22.04
RUN apt-get update && apt-get install -y libssl3
COPY --from=builder /app/target/release/quant-api /usr/local/bin/
CMD ["quant-api"]
```

#### 3.2.2 训练服务镜像

```dockerfile
# Dockerfile.training
FROM python:3.10-slim

WORKDIR /app
COPY python/requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY python/ .
CMD ["python", "train.py"]
```

### 3.3 Docker Compose

```yaml
version: '3.8'

services:
  api:
    build:
      context: .
      dockerfile: Dockerfile.api
    ports:
      - "8080:8080"
    volumes:
      - ./data:/app/data
      - ./config:/app/config
      - ./models:/app/models
    environment:
      - RUST_LOG=info
    restart: unless-stopped

  frontend:
    image: node:18-alpine
    working_dir: /app
    volumes:
      - ./frontend:/app
    ports:
      - "5173:5173"
    command: npm run preview

  training:
    build:
      context: .
      dockerfile: Dockerfile.training
    volumes:
      - ./data:/app/data
      - ./models:/app/models
    environment:
      - PYTHONUNBUFFERED=1
    profiles:
      - training

volumes:
  data:
```

### 3.4 启动Docker服务

```bash
# 构建镜像
docker-compose build

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 4. Jetson优化配置

### 4.1 性能模式设置

```bash
# 设置最大性能模式
sudo nvpmodel -m 0

# 设置最大时钟频率
sudo jetson_clocks
```

### 4.2 CUDA配置

```bash
# 验证CUDA安装
nvcc --version

# 设置环境变量
export CUDA_HOME=/usr/local/cuda
export PATH=$CUDA_HOME/bin:$PATH
export LD_LIBRARY_PATH=$CUDA_HOME/lib64:$LD_LIBRARY_PATH
```

### 4.3 内存优化

```bash
# 增加swap空间
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 永久生效
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

### 6.1 日志管理

**日志位置**:

- API日志: `logs/api.log`
- 训练日志: `logs/training.log`
- 系统日志: `/var/log/syslog`

**日志轮转**:

```bash
# 配置logrotate
sudo vim /etc/logrotate.d/quant

/path/to/quant-system/logs/*.log {
    daily
    rotate 30
    compress
    delaycompress
    missingok
    notifempty
}
```

## 7. 安全配置

### 7.1 防火墙设置

```bash
# 启用防火墙
sudo ufw enable

# 只允许本地访问
sudo ufw allow from 127.0.0.1 to any port 8080

# 或允许局域网访问
sudo ufw allow from 192.168.1.0/24 to any port 8080
```

### 7.2 API密钥管理

```bash
# 生成API密钥
openssl rand -hex 32 > .api_key

# 设置权限
chmod 600 .api_key
```

### 7.3 数据加密

```bash
# 加密敏感配置
gpg -c config/secrets.yaml
```

## 8. 故障排查

### 8.1 常见问题

**服务无法启动**:

- 检查端口占用: `lsof -i :8080`
- 检查日志: `journalctl -u quant-api`
- 检查配置文件

**内存不足**:

- 增加swap空间
- 优化模型大小
- 减少批处理大小

**推理延迟高**:

- 检查GPU使用: `nvidia-smi`
- 优化模型
- 调整线程数

### 8.2 性能调优

**监控工具**:

```bash
# CPU/内存监控
htop

# GPU监控
watch -n 1 nvidia-smi

# Jetson监控
tegrastats
```

**性能分析**:

```bash
# Rust性能分析
cargo flamegraph

# Python性能分析
python -m cProfile train.py
```

## 9. 升级和回滚

### 9.1 升级流程

```bash
# 1. 备份数据
./scripts/backup.sh

# 2. 拉取最新代码
git pull origin main

# 3. 重新构建
cargo build --release
npm run build

# 4. 重启服务
sudo systemctl restart quant-api
```

### 9.2 回滚流程

```bash
# 1. 切换到旧版本
git checkout v1.0.0

# 2. 重新构建
cargo build --release

# 3. 恢复数据(如需要)
./scripts/restore.sh backup_20260311.tar.gz

# 4. 重启服务
sudo systemctl restart quant-api
```
