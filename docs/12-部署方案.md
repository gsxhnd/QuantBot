# 部署方案

## 1. 本地部署（主要方案）

### 1.1 部署架构

- Rust 服务作为系统服务运行
- Vue 前端静态文件直接访问
- DuckDB 直接文件存储
- 无容器依赖

### 1.2 启动方式

```bash
# 启动 Rust API 服务
systemctl start quant-api

# 访问前端
http://localhost:8080
```

## 2. Docker 部署（可选）

### 2.1 适用场景

- 快速环境搭建
- 依赖隔离
- 备份迁移

### 2.2 架构特点

- 多阶段构建（Python 训练 + Rust 运行时）
- Docker Compose 编排
- 数据卷持久化
- ARM64 原生支持

### 2.3 目录结构

```
quant-system/
├── docker-compose.yml
├── Dockerfile
├── .env
└── data/
```

## 3. Jetson 优化

### 3.1 内存管理

- 可用内存：6-7GB
- 模型量化减少体积
- 数据冷热分离
- 定期垃圾回收
- 服务分离运行（训练/推理不并行）

### 3.2 存储优化

- DuckDB 列式压缩
- 索引优化
- 保留 2 年数据

### 3.3 计算优化

- CUDA 加速推理
- 推理吞吐：100-200 QPS
- P99 延迟：< 50ms
- 动态批处理

## 4. 性能目标

| 指标 | 目标值 |
|------|--------|
| 系统可用性 | 95% |
| 连续运行 | > 72h |
| API 响应 | < 2s |
| 推理延迟 | < 100ms |
