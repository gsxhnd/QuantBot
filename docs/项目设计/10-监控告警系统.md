# 监控告警系统

## 1. 监控维度

| 类别 | 监控指标 | 正常范围 | 告警触发 | 级别 |
|------|---------|---------|---------|------|
| **策略** | 今日收益率 | ±2% | 连续3天负收益 | WARNING |
| | 实时夏普比 | > 1.0 | < 0.5 | ERROR |
| | 胜率 | > 45% | < 35% | ERROR |
| **风险** | 实时回撤 | < 日均5% | > 日均150% | CRITICAL |
| | VaR突破 | 在阈值内 | 超历史95% | CRITICAL |
| | 头寸集中度 | < 5% | > 10% | WARNING |
| **价格** | 止损价触发 | 未触发 | 触发 | CRITICAL |
| | 止盈价触发 | 未触发 | 触发 | INFO |
| | 移动止损 | 未触发 | 触发 | WARNING |
| **数据** | 行情延迟 | < 1秒 | > 2秒 | WARNING |
| | 数据缺失 | 0% | 任何缺失 | CRITICAL |
| | 异常值占比 | < 0.5% | > 2% | ERROR |
| **系统** | CPU使用率 | < 70% | > 80% | WARNING |
| | 内存使用率 | < 75% | > 85% | ERROR |
| | 磁盘可用 | > 20% | < 10% | ERROR |

## 3. 止损止盈监控

### 3.1 用户设置监控条件

用户可为持仓设置价格监控条件：

```json
{
  "alert_config": {
    "stock_code": "AAPL",
    "position_id": "pos_001",
    "stop_loss": {
      "type": "price",
      "value": 180.00
    },
    "stop_loss_percent": {
      "type": "percent",
      "value": -0.05
    },
    "take_profit": {
      "type": "price",
      "value": 200.00
    },
    "trailing_stop": {
      "type": "trailing_percent",
      "value": 0.03
    }
  }
}
```

## 4. 异常检测算法

- **时间序列异常**：Isolation Forest + LOF混合
- **滑动窗口**：连续5日性能下降
- **统计异常**：Z-score > 3σ
- **梯度异常**：指标变化速率过快



|---------|---------|
| INFO | 记录，不通知 |
| WARNING | 普通通知 |
| ERROR | 即时通知 |
| CRITICAL | 紧急通知（电话/短信） |

## 6. 告警历史

所有告警记录存储到数据库，支持查询和分析：

```sql
SELECT * FROM alerts 
WHERE stock_code = 'AAPL' 
  AND created_at > '2024-01-01'
ORDER BY created_at DESC;
```
