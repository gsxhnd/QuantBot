# AI 模型设计

## 1. 模型架构

### 1.1 核心框架

**Python/Rust 混合架构**

- **Python 负责**: 数据获取、特征工程、模型训练、ONNX 导出
- **Rust 负责**: 模型加载、实时推理、交易信号生成

### 1.2 模型选择

**GRU (Gated Recurrent Unit) 时序模型**

选择理由:

- 适合时序数据预测
- 比 LSTM 更轻量,训练更快
- 在 Jetson 上推理性能更好
- 能够捕捉长期依赖关系

## 2. 训练流程

### 2.1 数据准备

#### 2.1.1 数据获取

- 从 yfinance 获取美股数据
- 从 Qlib 获取 A 股数据
- 数据范围: 至少 2 年历史数据

#### 2.1.2 数据清洗

- 处理缺失值
- 去除异常值
- 数据对齐

#### 2.1.3 特征工程

- 技术指标: MA, EMA, RSI, MACD, Bollinger Bands
- 价格特征: 收益率、波动率、振幅
- 成交量特征: 成交量变化率、量价关系
- 时间特征: 星期、月份、季度

### 2.2 模型定义

```python
# 模型结构示例
GRU(
  input_size=特征维度,
  hidden_size=128,
  num_layers=2,
  dropout=0.2,
  batch_first=True
) → Linear(hidden_size, 1)
```

**超参数**:

- 序列长度: 20 天
- 隐藏层大小: 128
- GRU 层数: 2
- Dropout: 0.2
- 学习率: 0.001
- 批大小: 64

### 2.3 训练配置

#### 2.3.1 损失函数

- MSE (Mean Squared Error) 用于回归
- 可选: Huber Loss 减少异常值影响

#### 2.3.2 优化器

- Adam 优化器
- 学习率衰减策略
- 梯度裁剪防止梯度爆炸

#### 2.3.3 早停机制

- 验证集损失连续 10 个 epoch 不下降则停止
- 保存最佳模型权重

### 2.4 训练执行

1. 数据集划分: 训练集 70%, 验证集 15%, 测试集 15%
2. 时间序列交叉验证防止前瞻偏差
3. GPU 加速训练
4. 记录训练日志和指标

### 2.5 模型评估

**评估指标**:

- MAE (Mean Absolute Error)
- RMSE (Root Mean Squared Error)
- R² Score
- 方向准确率(预测涨跌方向的准确率)

## 3. 推理实现

### 3.1 Rust 推理服务

#### 3.1.1 依赖配置

- `ort`: ONNX Runtime 绑定
- `ndarray`: 多维数组操作

#### 3.1.2 模型加载

```rust
// 创建推理会话
- 配置线程数优化
- 启用 CUDA 加速(如果可用)
- 设置内存优化选项
```

#### 3.1.3 推理执行

1. 接收实时特征数据
2. 数据预处理和标准化
3. 执行模型推理
4. 后处理预测结果
5. 返回预测值

### 3.2 信号生成

**交易信号逻辑**:

- 预测收益率 > 阈值 → 买入信号
- 预测收益率 < -阈值 → 卖出信号
- 其他 → 持有

**阈值设置**:

- 根据历史回测优化
- 考虑交易成本
- 动态调整
