# AI 模型设计

## 1. 模型架构

**Python/Rust 混合架构**
- Python：数据获取、特征工程、训练、ONNX 导出
- Rust：模型加载、实时推理、信号生成

**GRU 时序模型**
- 适合时序预测，比 LSTM 更轻量
- Jetson 推理性能好
- 捕捉长期依赖

## 2. 训练流程

### 2.1 数据准备

**数据源**：yfinance（美股）、Qlib（A股），至少 2 年历史

**特征工程**：
- 技术指标：MA、EMA、RSI、MACD、Bollinger Bands
- 价格特征：收益率、波动率、振幅
- 成交量特征：成交量变化率、量价关系
- 时间特征：星期、月份、季度

### 2.2 模型配置

**结构**：GRU(input → hidden_size=128, num_layers=2, dropout=0.2) → Linear(1)

**超参数**：
- 序列长度：20 天
- 学习率：0.001
- 批大小：64
- 优化器：Adam + 学习率衰减
- 早停：验证集损失 10 epoch 不降

**数据划分**：训练 70%、验证 15%、测试 15%

**评估指标**：MAE、RMSE、R²、方向准确率

## 3. 推理实现

**Rust 推理服务**（依赖：ort、ndarray）

**流程**：
1. 加载 ONNX 模型（配置线程、启用 CUDA）
2. 接收实时特征数据
3. 预处理和标准化
4. 执行推理
5. 生成交易信号

**信号逻辑**：
- 预测收益率 > 阈值 → 买入
- 预测收益率 < -阈值 → 卖出
- 其他 → 持有
- 阈值根据回测优化，考虑交易成本

## 4. Jetson 优化

| 优化层面 | 优化措施 |
|---------|---------|
| **模型** | INT8 量化、模型剪枝、知识蒸馏 |
| **运行时** | CUDA 加速、线程优化、内存池、批处理 |
| **系统** | 最大性能模式、调整频率、监控工具（tegrastats、nsys） |
| **部署** | ARM64 优化、预编译、特征缓存 |

## 5. 性能目标

| 指标 | 目标值 |
|------|--------|
| 推理延迟(P99) | < 50ms |
| 推理吞吐量 | > 100 QPS |
| 模型大小 | < 50MB |
| 内存占用 | < 500MB |
| 方向准确率 | > 55% |
